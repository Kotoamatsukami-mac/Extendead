# UI spec

## Primary window modes

### Lounge
Height: thin strip
Purpose: always-available command entry
Shape: rounded capsule, non-boxy
Visuals:
- dark glass base
- high contrast text
- restrained glow
- evil-rainbow accent only on focus, loading, approval, success edge
- compact left icon / status glyph
- central input
- compact right-side mode / pin controls

### Expanded
Purpose: inspection and controlled execution
Sections:
1. input strip
2. parsed intent chip
3. ambiguity route pills
4. risk + reversible badges
5. Y / N approval rail
6. live event timeline
7. result card
8. undo/history drawer
9. permission state

## Keybinds
- summon / hide
- toggle always-on-top
- expand / collapse
- focus input
- cancel current draft
